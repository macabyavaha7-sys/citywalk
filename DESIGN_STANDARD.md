# 神奇角落在哪里｜设计执行标准

本文件记录当前网站已经确认的设计理念、组件规则和更新流程。后续制作文章页、栏目页、详情页或其他组件时，应优先遵守这里的标准。

## 项目定位

- 项目名称：`L_One神奇角落在哪里`
- 网站主题：城市人文摄影、真实街头观察、图文与视频内容沉淀。
- 内容气质：真实、克制、有人味，不做营销式首页，不做过度装饰。
- 视觉方向：留白充足、信息清晰、移动端优先、内容窗口轻量、有阴影和圆角，但不过度圆润。

## 全站布局

- 顶部导航栏始终固定，不随滚动消失。
- 顶部导航栏之外，页面所有主要内容必须使用统一正文边距。
- 正文边距要保持宽松，不能贴边；手机端也要保留明确留白。
- 内容不能与视口边缘、导航栏或相邻模块发生重叠。
- 页面模块之间用空间和浅色分隔线区分，避免黑色硬边框。

## 顶部导航

- 中间品牌名称固定为：`L_One神奇角落在哪里`。
- 品牌文字使用粗黑体风格，居中显示。
- 左侧为隐藏下拉菜单，分类为：
  - 城市角落
  - 摄影技巧
  - 趣味瞬间
- 右侧为搜索框。
- 搜索框必须小巧，保持在右侧；窄屏时自动隐藏，避免遮挡品牌文字。

## 首页首屏

- 首屏由 slogan 和推荐内容窗口组成。
- slogan 固定为三行：
  - 钻进真实世界
  - 保持呼吸，无论什么味道
  - 保持体感，无论什么温度
- slogan 使用细黑体，不再使用粗黑体。
- slogan 字号需要随页面尺寸适配，但不得挤压或覆盖推荐窗口。
- slogan 与推荐窗口之间的间距应与正文边距保持一致。
- 推荐内容窗口为 `1:1` 正方形。
- 推荐窗口默认隐藏标题；鼠标悬停或划过封面时，标题从窗口底部出现。

## 作品展示

- 作品展示自适应设备宽度。
- 桌面最多 3 列，窄屏和手机保持 2 列。
- 作品区与窗口两侧保持统一正文留白，不能无限贴边放大。
- 每个作品封面为 `1:1` 方形图。
- 封面下方左侧标识内容类型：`图文` 或 `视频`。
- 封面下方右侧标识发布日期。
- 封面下方显示文章大标题和副标题。
- 首页最多展示 2 行，也就是 6 条内容。
- 内容超过 6 条时使用分页。

## 分页

- 分页按钮包括：`首页`、`上一页`、`下一页`、`尾页`。
- 手机端分页按钮必须缩小，始终保持一行。
- 分页按钮之间的间距保持稳定，不因按钮变大破坏留白。
- 按钮使用圆角和透明黑色阴影，不使用黑色实体边框。

## 联系方式

- 联系方式标题 `联系方式` 的视觉大小应与 `FIND L_ONE` 接近。
- 平台入口包括：小红书、抖音、微信公众号、bilibili、大众点评、马蜂窝等。
- 平台入口为 icon 在上、名称在下。
- 名称不自动换行，字号随 icon 一起保持紧凑。
- 手机端平台入口可横向滑动展示，避免挤压成大块。
- 邮箱、微信等二维码联系项尺寸应与平台 icon 卡片保持接近。
- 联系方式区域同样使用圆角和透明阴影，不使用黑色实体边框。

## 视觉组件

- 全站不使用黑色实体边框作为主要视觉框架。
- 原黑色边框统一改为透明度约 30% 的黑色阴影。
- 方形窗口需要圆角造型：
  - 推荐封面、作品封面：约 `8px`
  - 平台入口、小 icon、二维码卡片、下拉菜单：约 `6px`
- 圆角必须克制，不能变成过度圆润的卡通风格。
- 保留浅色分隔线用于模块划分，但避免厚重黑线。

## 数据与内容更新

- 首页内容数据集中维护在 `app.js`。
- 单篇文章页面完成后，优先通过更新 `app.js` 中的数据自动回填首页：
  - `featuredPost`
  - `posts`
  - `socialProfiles`
- 封面、标题、副标题、日期、类型、分类、链接都应通过结构化数据维护。
- 后续真实图片素材替换占位封面时，仍需保持 `1:1` 展示比例。

## GitHub 更新流程

- 当前远程仓库：`https://github.com/macabyavaha7-sys/citywalk.git`
- 默认分支：`main`
- 每次完成可验证改动后，应提交并推送到 GitHub，保证线上仓库持续更新。
- 推送前至少执行：
  - `node --check .\app.js`
  - 本地浏览器刷新预览关键页面
- 若涉及移动端样式，必须检查手机宽度下是否有重叠、溢出、换行异常。

## 后续开发默认执行习惯

- 开始新页面前，先参考本文件。
- 优先复用现有 CSS 变量、留白、阴影、圆角和响应式规则。
- 不新增与主页明显冲突的视觉语言。
- 新页面应与主页保持同一品牌气质：真实、清晰、克制、内容优先。
